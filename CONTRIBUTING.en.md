# Contributing

Languages: [Português (Brasil)](CONTRIBUTING.md) | **English** | [Español](CONTRIBUTING.es.md) | [Français](CONTRIBUTING.fr.md)

## Maintenance model

The AS-CNPJ ecosystem accepts contributions, but it operates with maintainer-led curation.

- external PRs are welcome;
- final merge decisions belong to the maintainers;
- proposals may be declined if they diverge from the ecosystem contract or strategy.

## Accepted contributions

- bug fixes;
- documentation improvements;
- new test vectors;
- test hardening;
- security hardening;
- build, CI and governance improvements.

## Changes that require prior alignment

Open an issue or discussion before sending a PR for:

- public API changes;
- algorithm changes;
- naming changes;
- license changes;
- new repositories in the family;
- structural governance changes.

## Review

Every PR requires review from at least one maintainer listed in `CODEOWNERS`.

## Security

Do not use public issues for vulnerabilities. See [SECURITY.md](SECURITY.md).
