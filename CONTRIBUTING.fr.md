# Contribution

Langues : [Português (Brasil)](CONTRIBUTING.md) | [English](CONTRIBUTING.en.md) | [Español](CONTRIBUTING.es.md) | **Français**

## Modèle de maintenance

L’écosystème AS-CNPJ accepte les contributions, mais fonctionne avec une gouvernance pilotée par le maintainer.

- les PR externes sont bienvenus ;
- la décision finale de fusion appartient aux maintainers ;
- une proposition peut être refusée si elle diverge du contrat ou de la stratégie de l’écosystème.

## Contributions acceptées

- corrections de bugs ;
- améliorations de documentation ;
- nouveaux vecteurs de test ;
- durcissement des tests ;
- durcissement de sécurité ;
- améliorations de build, CI et gouvernance.

## Changements nécessitant un alignement préalable

Ouvrez une issue ou une discussion avant d’envoyer un PR pour :

- des changements d’API publique ;
- des changements d’algorithme ;
- des changements de naming ;
- des changements de licence ;
- de nouveaux dépôts dans la famille ;
- des changements structurels de gouvernance.

## Revue

Chaque PR requiert la revue d’au moins un maintainer listé dans `CODEOWNERS`.

## Sécurité

N’utilisez pas d’issue publique pour les vulnérabilités. Voir [SECURITY.md](SECURITY.md).
