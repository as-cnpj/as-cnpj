# Contribuindo

Idiomas: **Português (Brasil)** | [English](CONTRIBUTING.en.md) | [Español](CONTRIBUTING.es.md) | [Français](CONTRIBUTING.fr.md)

## Modelo de Manutenção

O `as-cnpj-js` aceita contribuições, mas opera com curadoria central do maintainer.

- PRs externos são bem-vindos;
- a decisão final de merge fica com os maintainers;
- uma proposta pode ser rejeitada se divergir do contrato da biblioteca ou da estratégia do ecossistema.

## Tipos de Contribuição Aceitos

- correção de bug;
- melhoria de documentação;
- novos vetores de teste;
- hardening de testes;
- hardening de segurança;
- melhorias de build, CI e release;
- melhorias de ergonomia sem quebrar o contrato público.

## Mudanças que Exigem Alinhamento Prévio

Abra issue ou discussion antes de enviar PR para:

- mudança de API pública;
- mudança do algoritmo;
- mudança de naming;
- mudança de licença;
- mudança estrutural do repositório.

## Review

Todo PR precisa de review de pelo menos um maintainer listado em `CODEOWNERS`.

## Segurança

Não use issue pública para vulnerabilidades. Consulte [SECURITY.md](SECURITY.md).
