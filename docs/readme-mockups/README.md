# Mockups de README para `as-cnpj-js`

Este diretório reúne propostas de README em estilos diferentes para o repositório `as-cnpj-js`.

O objetivo não é aprovar texto final linha a linha agora, e sim escolher:

- qual estrutura comunica melhor o projeto;
- qual profundidade faz mais sentido para o público;
- qual tom deve virar o padrão dos próximos repositórios da org.

## Modelos

### 1. Modelo A: Executivo e confiável

Arquivo: [as-cnpj-js-modelo-a-executivo.md](as-cnpj-js-modelo-a-executivo.md)

Perfil:

- mais corporativo;
- abre com posicionamento, escopo, garantias e quick links;
- bom para transmitir seriedade, clareza e confiança.

Quando usar:

- se a prioridade for credibilidade institucional;
- se o público inclui engenharia, arquitetura e gestão técnica.

### 2. Modelo B: Dev-first e orientado a adoção

Arquivo: [as-cnpj-js-modelo-b-dev-first.md](as-cnpj-js-modelo-b-dev-first.md)

Perfil:

- vai rápido para uso, API e exemplos;
- reduz fricção para quem quer testar a biblioteca imediatamente;
- deixa o README mais “mão na massa”.

Quando usar:

- se a prioridade for conversão de uso;
- se o público principal for dev que quer copiar, testar e integrar.

### 3. Modelo C: Produto e ecossistema

Arquivo: [as-cnpj-js-modelo-c-ecossistema.md](as-cnpj-js-modelo-c-ecossistema.md)

Perfil:

- conecta fortemente a biblioteca ao hub e à tese do projeto;
- apresenta melhor a org e a consistência entre linguagens;
- mais estratégico e mais “marca de ecossistema”.

Quando usar:

- se você quiser que cada repo reforce a identidade da org;
- se o README também for peça de posicionamento público.

## Minha leitura

Se o objetivo for ter um README principal forte para o `as-cnpj-js`, eu recomendo:

1. usar o **Modelo A** como base;
2. puxar do **Modelo B** a ordem de “comece por aqui” e os exemplos curtos;
3. puxar do **Modelo C** apenas a parte de ecossistema.

Em termos práticos:

- `Modelo A` tende a ser o melhor README oficial;
- `Modelo B` tende a ser o melhor README para adoção rápida;
- `Modelo C` tende a ser o melhor README para branding do ecossistema.
